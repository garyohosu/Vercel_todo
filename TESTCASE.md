# テストケース定義

## テスト方針

| レベル | ツール（予定） | 対象 |
|--------|--------------|------|
| 単体テスト | Jest / Vitest | TodoRepository・バリデーション関数・ソート関数 |
| 統合テスト | Jest + Neon テスト DB | API Route Handler |
| E2E テスト | Playwright（後回し） | 画面操作フロー |

---

## TC-01: タスク一覧表示

### 正常系

| # | 前提条件 | 操作 | 期待結果 |
|---|----------|------|----------|
| 1-1 | タスクが複数存在する | GET /api/todos | 全件返却される |
| 1-2 | 未完了と完了タスクが混在 | 一覧表示 | 未完了が上・完了が下に表示される |
| 1-3 | due_date あり・なし混在 | 一覧表示 | due_date あり → 昇順、null → 末尾に表示される |
| 1-4 | due_date が同値のタスクが複数 | 一覧表示 | created_at 降順（新しい順）で表示される |
| 1-5 | タスクが0件 | 一覧表示 | 「タスクがありません」メッセージが表示される |
| 1-6 | due_date 未設定タスクあり | 一覧表示 | 締切日列に「─」が表示される |
| 1-7 | 任意のタスクあり | 一覧表示 | 各行に作成日時が YYYY/MM/DD HH:mm 形式で表示される |

### 異常系

| # | 前提条件 | 操作 | 期待結果 |
|---|----------|------|----------|
| 1-8 | DB 接続エラー | GET /api/todos | 500 エラーレスポンスが返る |
| 1-9 | GET /api/todos が失敗 | 一覧表示 | 再試行導線付きエラーメッセージが一覧領域に表示される |

## TC-02: タスク追加

### 正常系

| # | 入力 | 期待結果 |
|---|------|----------|
| 2-1 | title="買い物", 他省略 | 201 Created、DB にレコード作成、status=todo、priority=medium |
| 2-2 | title + description + dueDate + priority="high" | 201 Created、全項目が保存される |
| 2-3 | title 200文字ちょうど | 201 Created |

### バリデーション（フロント）

| # | 入力 | 期待結果 |
|---|------|----------|
| 2-4 | title="" | API を呼ばずエラーメッセージ表示 |
| 2-5 | title=201文字 | API を呼ばずエラーメッセージ表示 |
| 2-6 | description=1001文字 | API を呼ばずエラーメッセージ表示 |
| 2-7 | dueDate="2024-13-01"（不正な日付） | API を呼ばずエラーメッセージ表示 |

### バリデーション（API）

| # | リクエスト | 期待結果 |
|---|----------|----------|
| 2-8 | POST /api/todos (title="") | 400 Bad Request |
| 2-9 | POST /api/todos (title なし) | 400 Bad Request |
| 2-10 | POST /api/todos (dueDate="2024-13-01") | 400 Bad Request |

## TC-04: タスク編集

### 正常系

| # | 前提条件 | 操作 | 期待結果 |
|---|----------|------|----------|
| 4-1 | 既存タスク | 編集ボタンをクリック | 編集画面に遷移し、既存データが入力欄に表示される |
| 4-2 | 編集画面でタスク名を変更 | 保存する | DB が更新され、一覧ページに戻る |
| 4-3 | 編集画面でキャンセル | キャンセルボタン | DB は変更されず、一覧ページに戻る |

### バリデーション（フロント）

| # | 入力 | 期待結果 |
|---|------|----------|
| 4-4 | title="" | API を呼ばずエラーメッセージ表示 |

### バリデーション（API）

| # | リクエスト | 期待結果 |
|---|----------|----------|
| 4-6 | PUT /api/todos/:id (title="") | 400 Bad Request |
| 4-7 | PUT /api/todos/:id (description=1001文字) | 400 Bad Request |
| 4-8 | PUT /api/todos/:id (dueDate="2024-13-01") | 400 Bad Request |

### 異常系

| # | 前提条件 | 操作 | 期待結果 |
|---|----------|------|----------|
| 4-5 | 存在しない ID | /todos/999/edit にアクセス | 「タスクが見つかりません」を表示し、一覧への戻りリンクを表示する |
| 4-9 | GET /api/todos/:id が 500 を返す | /todos/:id/edit にアクセス | エラーメッセージを表示し、フォームは表示しない |
| 4-10 | PUT /api/todos/:id が失敗 | 保存する | フォーム上部にエラー表示、入力内容は保持、一覧に戻らない |

## TC-06: 絞り込み表示

| # | 選択フィルター | 前提条件 | 期待結果 |
|---|--------------|----------|----------|
| 6-1 | すべて | 複数状態のタスクが存在 | 全件表示される |
| 6-2 | 未着手 | todo/doing/done が混在 | status=todo のみ表示される |
| 6-3 | 進行中 | todo/doing/done が混在 | status=doing のみ表示される |
| 6-4 | 完了 | todo/doing/done が混在 | status=done のみ表示される |
| 6-5 | 期限切れ | due_date < 今日 & 未完了のタスクあり | 該当タスクのみ表示される |
| 6-6 | 今日まで | due_date <= 今日 & 未完了のタスクあり | 該当タスクのみ表示される |
| 6-7 | 高優先度 | priority=high のタスクあり | priority=high のみ表示される |
| 6-8 | 未着手（0件） | status=todo のタスクなし | 「タスクがありません」が表示される |
| 6-9 | 期限切れ | due_date<今日 かつ status=done のタスクあり | done タスクは表示されない |
| 6-10 | 今日まで | dueDate=null のタスクあり | dueDate 未設定タスクは表示されない |
