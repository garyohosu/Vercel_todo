アプリの目的

日々のやることを登録し、進捗を管理するためのシンプルなWebアプリ。

想定する使い方はこんな感じ。

やることを追加する
終わったら完了にする
不要になったら削除する
未完了タスクを優先して見る
締切が近いものを確認する
開発方針
最初の版で重視すること
小さく作れる
GitHub連携しやすい
Vercelに載せやすい
NeonでDB保存できる
あとで機能追加しやすい
最初の版で入れないもの
ログイン機能
複数ユーザー対応
通知メール
ファイル添付
複雑なラベル管理
サブタスク
ドラッグ＆ドロップ並べ替え

このへんは後で増築できる。最初から全部入れると、家を建てる前に地下迷宮を掘り始める感じになる。

システム構成
フロント: Next.js
バックエンドAPI: Next.js Route Handler
DB: Neon(PostgreSQL)
ソース管理: GitHub
デプロイ: Vercel
機能仕様
1. タスク一覧表示

登録済みのタスクを一覧表示する。

表示項目:

タスク名
説明
締切日
優先度
状態
作成日時

並び順:

未完了を上
その中で締切が近い順
完了済みは下
2. タスク追加

新しいタスクを登録できる。

入力項目:

タスク名（必須）
説明（任意）
締切日（任意）
優先度（低・中・高）
状態（初期値は未着手）

バリデーション:

タスク名は1文字以上
長すぎる入力は制限する
締切日は不正な日付を不可にする
3. タスク状態変更

タスクの状態を変更できる。

状態:

未着手
進行中
完了

操作例:

一覧からワンクリックで完了
完了済みを未着手に戻す
編集画面で進行中に変更
4. タスク編集

登録済みタスクの内容を修正できる。

編集可能項目:

タスク名
説明
締切日
優先度
状態
5. タスク削除

不要なタスクを削除できる。

仕様:

削除前に確認メッセージを出す
物理削除でよい
最初は論理削除までいらない
6. 絞り込み表示

一覧を条件で絞り込める。

絞り込み条件:

すべて
未着手
進行中
完了
期限切れ
今日まで
高優先度

これはかなり便利。ToDoアプリは「入れる」より「探す」で困ることが多いからね。

7. 検索

タスク名や説明で検索できる。

検索対象:

タスク名
説明

最初は部分一致で十分。

画面仕様
1. 一覧画面

メイン画面。

表示内容:

上部にタイトル
タスク追加フォーム
絞り込みボタン
検索ボックス
タスク一覧

各タスク行に置く操作:

完了切替
編集
削除
2. 編集画面

別ページでもモーダルでもいいけど、最初は別ページがおすすめ。
理由は単純で、実装がわかりやすいから。

データベース仕様

テーブル名は todos でいいと思う。

カラム案
id
title
description
status
priority
due_date
created_at
updated_at
各項目の意味
id: タスクID
title: タスク名
description: 詳細説明
status: 未着手 / 進行中 / 完了
priority: 低 / 中 / 高
due_date: 締切日
created_at: 作成日時
updated_at: 更新日時
ステータス仕様

状態の値は、最初から日本語文字列でもいいけど、DBには英語の固定値の方が扱いやすい。

例:

todo
doing
done

優先度も同じ。

例:

low
medium
high

画面では日本語表示に変換する。

API仕様
1. タスク一覧取得

GET /api/todos

用途:

一覧表示（v1 は全件取得のみ）

備考:

絞り込み・検索はフロントエンド側で行う（v1 ではクエリパラメーターを使わない）
2. タスク追加

POST /api/todos

用途:

新規作成

送信例:

title
description
dueDate
priority
status
3. タスク更新

PUT /api/todos/:id

用途:

編集
状態変更
4. タスク削除

DELETE /api/todos/:id

用途:

削除
業務ルール
完了済みタスクも一覧には残す
締切が過ぎて未完了なら「期限切れ」と表示する
締切未設定のタスクも登録可能
タスク名が空なら登録不可
同名タスクは登録可
同じ名前の作業が別の日に発生することは普通にあるから
非機能要件
性能
個人利用を想定
数百件程度のタスクを快適に扱えること
セキュリティ
初版は個人用なので認証なしでも可
ただし公開URLの扱いには注意
将来的には認証追加可能な構成にする
保守性
フロントとDBアクセスの責務を分ける
ステータス値は定数化する
後でユーザーIDを追加しやすい設計にする
将来拡張案
